# Regenerate and check

Use this loop after any descriptor or catalog edit:

```bash
go run ./cmd/axle gen --descriptor descriptors/<resource>/descriptor.axle.json --out descriptors/<resource>/generated --json
go run ./cmd/axle catalog gen --manifest catalog/axle.catalog.json --out catalog --json
go mod tidy
go run ./cmd/axle gen --descriptor descriptors/<resource>/descriptor.axle.json --out descriptors/<resource>/generated --check --json
go run ./cmd/axle catalog gen --manifest catalog/axle.catalog.json --out catalog --check --json
go run ./cmd/axle check --root . --json
go test ./...
go vet ./...
```

Generated files carry `Code generated by axle gen; DO NOT EDIT.` Do not patch generated files manually; change the descriptor or catalog manifest and regenerate.

For a generated backend scaffold, run `./scripts/verify.sh`; it includes descriptor checks, catalog checks, architecture checks, tests, vet, and build.
The scaffold verify script regenerates descriptor and catalog output before
`go mod tidy` so module resolution does not fail on stale generated imports
after a resource rename.
